## 5 层架构总览

```
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 0: RSS 采集（已有，每30min cron）                              │
│  ─────────────────────────────────────────                          │
│  94个RSS源 → rss-scanner.py → SQLite (rss-archive.db)               │
│  产出: 标题 + 200字摘要 + URL + 分类                                 │
│  成本: 0（纯Python，🈚 大模型）                                       │
└──────────────────────────┬──────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 1: 重要程度判断（策略选择层）                                   │
│  ────────────────────────────────────                               │
│  方式A（当前）: SQL 查询过滤 category/domain/关键词                    │
│  方式B（可选）: 🤖 DeepSeek 批量打分 (prompt: "这是财经AI新闻，重要吗？")│
│                └─ 仅当需要语义理解时触发，批量送20条/次，cost≈$0.001   │
│  产出: 待抓取 URL 列表 (urls.txt)                                    │
└──────────────────────────┬──────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 2: 级联抓取（python batch.py，🈚 大模型）                       │
│  ─────────────────────────────────────────                          │
│                                                                     │
│  对于每个URL:                                                        │
│                                                                     │
│  ┌──────────────────────────────────────────┐                       │
│  │ 域名画像查表 (domain_profiles.py)         │ ← 17域名知识库          │
│  │  ├─ bloomberg.com → anti_bot=datadome     │                       │
│  │  │   order: [direct, google_cache,        │                       │
│  │  │           archive, search_snippet]     │                       │
│  │  ├─ bbc.co.uk       → anti_bot=none       │                       │
│  │  │   order: [direct]                      │                       │
│  │  └─ 未知            → 全梯度降级           │                       │
│  └──────────────┬───────────────────────────┘                       │
│                 │                                                   │
│                 ▼                                                   │
│  ┌──────────────────────────────────────────┐                       │
│  │ 代理路由 (_make_client)                   │                       │
│  │  国内域名(.cn) → 直连                     │                       │
│  │  境外域名      → 127.0.0.1:10808          │                       │
│  └──────────────┬───────────────────────────┘                       │
│                 │                                                   │
│                 ▼                                                   │
│  ┌──────────────────────────────────────────┐                       │
│  │ 成本感知级联 (cascade.py)                  │                       │
│  │                                          │                       │
│  │  direct ──❌──► google_cache ──❌──►       │                       │
│  │  archive ──❌──► search_snippet           │                       │
│  │                                          │                       │
│  │  成功即停，不浪费后续调用                    │                       │
│  │  每次尝试记录 cost_trace                   │                       │
│  └──────────────┬───────────────────────────┘                       │
│                 │                                                   │
│                 ▼                                                   │
│  ┌──────────────────────────────────────────┐                       │
│  │ RateLimiter 限速                          │                       │
│  │  同域名最小间隔 1s                         │                       │
│  │  友好域名(reuters) 0.5s                   │                       │
│  │  browser策略 额外 5s 冷却                  │                       │
│  └──────────────┬───────────────────────────┘                       │
│                 │                                                   │
│                 ▼                                                   │
│  ┌──────────────────────────────────────────┐                       │
│  │ 正文提取 (fetchers.py)                     │                       │
│  │                                          │                       │
│  │  策略          实现         成本           │                       │
│  │  ──────────────────────────────           │                       │
│  │  direct     httpx+trafilatura   1         │                       │
│  │  google_cache webcache.google   1         │                       │
│  │  archive    web.archive.org     1         │                       │
│  │  scrapling  StealthyFetcher     2         │                       │
│  │  browser    Playwright headless  3         │                       │
│  └──────────────────────────────────────────┘                       │
│                                                                     │
│  并发: ThreadPoolExecutor(max_workers=4)                             │
│  输出: JSONL (每行一条，含 正文/cost_trace/temporal_check)            │
│  成本: 0（🈚 大模型）                                                │
└──────────────────────────┬──────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 3: 时间校验 (temporal.py，🈚 大模型)                            │
│  ─────────────────────────────────────────                          │
│  五条硬规则:                                                         │
│    1. URL年份 vs 标题年份 → 冲突检测                                   │
│    2. 发布年份 vs 当前年份 → 时效性标记                                 │
│    3. 多源交叉验证 → 一致性评分                                        │
│  产出: confidence(high/medium/low) + conflicts[]                     │

                           │
                           ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 4: 🤖 结构化抽取（可选，仅 --llm-extract 时触发）               │
│  ─────────────────────────────────────────                          │
│                                                                     │
│  DeepSeek API (deepseek-chat)                                       │
│  输入: 正文前 8000 字符 + JSON Schema prompt                         │
│  输出: {"headline","published_at","summary","key_points":[...]}     │
│  成本: ~$0.002/篇                                                    │
│  关键: 一篇文章只调一次，结果落 JSONL 缓存，不重复调用                   │
│                                                                     │
│  执行位置: batch.py 的 extract_url() 在正文抓取成功后                   │
│           if llm_api_key and llm_prompt:                            │
│               structured = llm_extract_structured(...)              │
└──────────────────────────┬──────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 5: 🤖 多源融合（高级场景，仅A级breaking事件）                    │
│  ─────────────────────────────────────────                          │
│                                                                     │
│  s06_merge.py — 同一事件多篇文章交叉验证：                              │
│    输入: 2+ 篇文章正文                                                │
│    输出: {consensus_facts, divergent_claims, confidence}             │
│    成本: ~$0.005/次                                                  │
│    触发条件: A级事件 (breaking news) 且 require_cross_check=true      │
└──────────────────────────┬──────────────────────────────────────────┘
                           │
                           ▼
                    ┌──────────┐
                    │ results  │  ← JSONL 最终输出
                    │ .jsonl   │    每行: {url, content, structured?,
                    └──────────┘            temporal_check, cost_trace}
```

## 🤖 大模型使用环节总览

| 层 | 环节 | 触发方式 | 模型 | 频率 | 成本/篇 |
|:--:|------|------|------|:--:|:--:|
| **1** | 重要程度语义打分 | 可选，SQL关键词过滤够用时跳过 | DeepSeek | 批量20条/次 | ~$0.001 |
| **2** | 级联抓取 | 🚫 **不用大模型** | — | — | **$0** |
| **3** | 时间校验 | 🚫 **不用大模型**（纯算法） | — | — | **$0** |
| **4** | 结构化JSON抽取 | `--llm-extract`，正文获取后调一次 | DeepSeek | 1次/篇 | ~$0.002 |
| **5** | 多源交叉融合 | A级breaking事件 + 手动触发 | DeepSeek | 按需 | ~$0.005 |

## 2026-07-08 首次实战结果

```bash
$ python batch.py --urls rss-test-urls.txt --out rss-results.jsonl --verbose

完成: 2/3 成功, 1 失败
总耗时: 4.3s | 总成本: 2 | 速率: 0.7 urls/s
```

| 来源 | 域名 | cascade | 字数 | 状态 |
|------|------|---------|------|------|
| BBC | bbc.co.uk | direct✅ | 3,695 | ✅ |
| Al Jazeera | aljazeera.com | direct✅ | 4,454 | ✅ |
| Bloomberg | bloomberg.com | direct→403, archive→404 | — | ❌ 当日未归档 + 无Google Cache网络 |

### 实战中修复的问题

1. **Archive URL 格式**：`/web/{url}` → `/web/0/{url}`（缺少斜杠导致404）
2. **代理自动分流**：`_make_client` 新增 `_is_chinese_domain()` 判断，境外走10808、国内直连
3. **bbc.co.uk 域名画像**：补充到 domain_profiles（原只有 bbc.com 导致匹配失败）
4. **Google Cache 策略**：新增 `fetch_google_cache()`，加入 Bloomberg/WSJ/FT 策略链
