✅ Phase 0 → 1: RSS Scanner → rss-archive.db

Python独立进程 · 每30min cron触发 · 输出: ~/.hermes/rss-archive.db (rss_articles表)

字段类型来源 / 备注示例值
sourceTEXT RSS feed 名称 Reuters, DW News, BBC
titleTEXT feedparser entry.title Apple calls OpenAI's hardware business 'rotten...'
summaryTEXT feedparser entry.summary (HTML stripped) Apple has accused the company behind ChatGPT...
linkTEXT feedparser entry.link → 唯一键 https://www.dw.com/en/apple-sues-openai/a-123456
categoryTEXT RSS feed 原生分类标签 World, Technology, Business
dateTEXT feedparser entry.published_parsed 2026-07-10T08:30:00Z
📤 输出 → Phase 1 sync.py 读取: source, title, summary, link, category, date

✅ Phase 1.1: sync.py + scorer.py → news_intel.db (评分)

sync.py 读取 rss-archive.db → 调 scorer.score_article() → 写入 news_intel.db 三表

📥 输入 (sync_recent 读取)

输入字段来自用途
sourcerss-archive.db→ source_name 传给 score_source() 查权威度
titlerss-archive.db→ 传给 score_impact() + compute_velocity()
summaryrss-archive.db→ description 传给 score_impact() → 关键词匹配
linkrss-archive.db→ article_url 写入 rss_raw + URL指纹去重
categoryrss-archive.db→ category_raw 写入 rss_raw
daterss-archive.db→ published_at 写入 rss_raw

score_article() 五维评分输出

字段分数区间评分逻辑示例
total0–100五维: source+impact+entity+market+velocity92
source0–20查表: source_scores.jsonUK Gov=20, Reuters=18, DW News=16, 未知=5
impact0–30event_keywords.json 关键词正则命中战争/制裁/ceasefire → 30, 财报 → 15
entity0–20entity_weights.json 实体权重查表US Federal Reserve=20, 小公司=3
market0–20asset_graph.json 市场关联匹配NVDA→半导体=18, 无关=0
velocity0–10compute_velocity() ±30min窗口词集Jaccard相似度5篇同事件=10, 孤立=0

score_article() 额外输出

字段类型说明
tierTEXT'A'(>90) | 'B'(60-90) | 'C'(<60)
categorieslist[str]关键词命中的分类: e.g. [Military, Diplomatic, Economic]
entitiesdict{companies:[], persons:[], countries:[]} — entity_weights.json 查表提取
market_assetslist[str]asset_graph.json 匹配到的资产/股票代码
velocity_countint同期(Jaccard≥0.5) 内同事件报道数

📤 写入 news_intel.db 三表

→ rss_raw (18字段)
guid(link), source_name, source_domain, article_url, title, description, published_at, category_raw, …
→ news_intelligence (14字段)
raw_id, score_total, score_source, score_impact, score_entity, score_market, score_velocity, tier, category, tags(JSON), entities(JSON), velocity_count, scored_at
→ news_content (本阶段仅占位，Phase 2填充)
intel_id, article_url, content_md, … (见 Phase 2 输出)

✅ Phase 3: aggregator.py v4.3 (事件聚合 · 零LLM)

读 news_intel.db → build_fingerprint → cluster → merge → 输出21字段 Event Object

📥 输入 (test_aggregator.py 跨三表 JOIN)

输入字段来自表聚合中使用
idnews_contentarticle_ids 追溯
titlerss_raw→ _detect_action() 动作检测 → 计数排序
summary_cnnews_content→ _classify_topics() 话题分类 → 摘要拼接
descriptionrss_raw→ _get_text() 全文检索 → topic + action detail
score_totalnews_intelligence→ 用于排序 (高分文章优先作为 centroid)
tiernews_intelligence→ 参考 (仅A/B级参与)
entitiesnews_intelligence→ build_fingerprint 的 Subject/Object 来源 (JSON: companies/persons/countries)
published_atrss_raw→ 时间窗过滤 + stage 计算
source_namerss_raw→ source.primary_source + source.sources[]

内部指纹结构 build_fingerprint()

字段类型生成逻辑
subjectstrcompanies/persons 中 entity_weight 最大的 (hub×0.3, ≥MIN_SUBJECT_WEIGHT=0.15)
subject_weightfloatglobal_IDF × topic_IDF × entity_type_weight 的最终值
actionstr14类动作枚举: ATTACKS|DIES|SUES|NEGOTIATES|… → 计数排序取最多
objectstrcountries/companies 中加权最高 (不含 subject, hub×0.3)
object_weightfloat同 subject_weight 逻辑
event_typestrACTION_MAP[action][0]: Military|Legal|Political|Diplomatic|…
primary_topicstrTOPIC_SIGNALS 命中数最多的分类
secondary_topicstrTOPIC_SIGNALS 命中数次多的分类
countrystr|Noneentities.countries[0] canonicalized → location 硬约束
participantsfrozenset_extract_participants(): entities.countries + COUNTRIES正则
anchorstr{subject}|{action}|{object}|{primary} → 快速完全匹配捷径

fingerprint_score() 评分维度

维度分数条件
location 硬约束return 0不同 country → 直接 0 分 (硬拒绝)
anchor 快速通道return 100完全相同的 SAO+Topic → 立即合并
Action25同动作类型 (非OTHER)
Subject10~25同名 + 稀有度加权 (subject_weight越高分越高)
Object10~30同名 + 稀有度加权
Topic10primary 相同=10, secondary相同=5
Event Type10同 event_type
Participants bonus5~10≥2个国家重叠=10, 1个=5
阈值≥50→事件 | ≥75→强合并 |

📤 输出: 21字段 Event Object

(参考 SCHEMA.md 和架构图 — 完整 JSON Schema 已在上轮给出)

字段类型生成逻辑
event_idstrEVT-{YYYYMMDD}-{idx+1:03d}
subject{name,type}objfingerprint centroid → _known_entity_types 查类型
action{type,detail}objfingerprint centroid + _extract_action_detail() 正则后50字符
object{name,type}objfingerprint centroid → _known_entity_types
event_typestrfingerprint centroid
event_timestr|nullev.start_time.isoformat()
location{country,region}objfingerprint.country + region(null 预留)
source{primary,authority,count,sources[]}objfirst_article.source_name + max权威度 + 去重计数
doc_refs[{url,title}]list前5篇成员 article_url + title
actors[{entity,type,role}]list_infer_actor_roles(): #1=Initiator, obj_match=Target, 其余=Participant
titlestrev.best_title (合并cluster中最描述性的)
summarystr前3篇 summary_cn/description 拼接 (HTML过滤, <30%标签)
keywordslistev.topics 聚合
confidencefloat0.4×auth/20 + 0.3×coherence/100 + 0.2×diversity + 0.1×volume
coherencefloat成员间平均 pairwise 相似度
extraction_methodstr固定: "v4.3-saeo"
related_entities[{name,type}]list前20个 entity_refs
article_countintlen(ev.article_ids)
article_idslist[int]ev.article_ids (DB主键)
stagestr距 first_seen: <2h=breaking | 2-24h=developing | 24h-7d=active | 7-30d=stable | >30d=closed
first_seenstr|nullev.start_time.isoformat()
last_updatedstr|nullev.last_time.isoformat()
