News Intelligence Platform — Phase 1 Architecture

RSS Ingestion → Search Engine v2 → Entity Scoring → Event Aggregation → Frozen Schema Output

Data Pipeline Stats

  • • 46 RSS + 18 Nitter + 6 Chinese sources
  • • Scan every 30min via cron (SOCKS5 proxy)
  • • URL dedup by SHA256 fingerprint
  • • Qwen3 classification: topic + entities + NER
  • • Cost: ~$0.02 per classification call

Search Engine v2

  • • 16 domain profiles (WSJ→BBC→NYT→CNN)
  • • 4-step cost cascade: direct→archive→Scrapling→playwright
  • • Temporal V2 with 5 hard validation rules
  • • Rate limiter: per-domain token bucket
  • • Pure Python — NO Agent/LLM in retrieval

Aggregation Pipeline

  • • Zero-LLM: rule based, deterministic
  • • SAO extraction + canonicalization
  • • Hub dampening (×0.3 for >15% entities)
  • • Location hard constraint (different country→0)
  • • Confidence: source_auth + coherence + diversity

Output Schema (21 fields)

  • • event_id · subject/action/object structured
  • • source authority · actors · confidence
  • • lifecycle stage (breaking→closed)
  • • doc_refs · keywords · summary
  • • Cost: 1 Qwen3 call per article (qmerge)
