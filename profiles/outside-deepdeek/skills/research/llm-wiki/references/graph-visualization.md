# Graph Visualization for LLM Wiki

## Overview

Two visualization modes exist for the wiki's knowledge graph:

| Mode | Requires | Best For |
|------|----------|----------|
| Obsidian Graph View | Obsidian app | Interactive exploration, daily use |
| HTML Graph (D3.js) | Any browser | Sharing, embedding, quick glance |

## Obsidian Graph View

Open the wiki directory as an Obsidian vault:
```bash
# Windows
start "D:\Program Files\Obsidian\Obsidian.exe" "obsidian://open?vault=C:\Users\You\wiki"
```

Keyboard shortcuts:
- `Ctrl+G` — Global Graph (all pages)
- `Ctrl+Shift+G` — Local Graph (current page + direct connections)

### Graph Controls in Obsidian
- **Filters**: Exclude tags, search terms
- **Groups**: Color-code by tag or folder
- **Depth**: 1-3 hops for local graph

## HTML Interactive Graph (D3.js)

A standalone HTML file that renders the wiki as a force-directed graph.
No Obsidian or server required — open `graph.html` in any browser.

### How It Works

1. **Data source**: `data.json` at the wiki root, generated by `scripts/wiki-graph.py`
2. **Renderer**: `graph.html` with embedded D3.js v7 (loaded from CDN)
3. **Extraction**: The Python script scans all `.md` files (except raw/),
   parses YAML frontmatter for type/tags, and extracts `[[wikilinks]]`

### Setup

**Step 1: Create `scripts/wiki-graph.py`**

This script walks `entities/`, `concepts/`, `comparisons/`, `queries/`,
parses frontmatter (title, type, tags), extracts `[[wikilinks]]`, and
outputs `data.json` with nodes and edges.

Key algorithm:
```python
for fpath in WIKI_DIR.rglob("*.md"):
    if relpath in EXCLUDED or relpath.startswith(("raw/", "_archive/", ".obsidian/")):
        continue
    content = fpath.read_text(encoding="utf-8")
    fm = parse_frontmatter(content)      # title, type, tags
    links = extract_wikilinks(content)   # [[Page Name]] → list
    files[relpath] = {
        "title": name,
        "frontmatter": fm,
        "links": links,
        "size": len(content),
    }
# Resolve links by fuzzy-matching [[link]] names to page slugs
# Build edges: {source: slugA, target: slugB}
```

**Step 2: Create `graph.html`**

Self-contained HTML with:
- Dark theme matching Obsidian aesthetic
- D3.js force-directed layout (charge, link distance, collision)
- Color coding by type (entity=purple, concept=blue, comparison=amber, query=green)
- Interactive: drag, zoom, pan, hover tooltips, search filter
- Legend overlay

Dependencies: D3.js v7 from CDN (`https://d3js.org/d3.v7.min.js`).

**Step 3: Generate and view**
```bash
cd /path/to/wiki
python scripts/wiki-graph.py
start graph.html    # Windows
open graph.html     # macOS
xdg-open graph.html # Linux
```

### When to Rebuild

- After any wiki page create/update/delete
- Before viewing graph.html
- The auto-sync script should always rebuild before push

## Graph Data Format (data.json)

```json
{
  "files": {
    "entities/transformer.md": {
      "title": "Transformer Architecture",
      "frontmatter": { "type": "entity", "tags": ["deep-learning"], "confidence": "high" },
      "links": ["Attention Mechanism", "BERT", "GPT"],
      "size": 3200
    }
  },
  "stats": {
    "total_pages": 42,
    "nodes_with_type": { "entity": 15, "concept": 18, "comparison": 5, "query": 4, "other": 0 }
  }
}
```
